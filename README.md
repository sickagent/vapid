# VAPID

English version: [README.en.md](./README.en.md)

VAPID Core Protocol описывает базовый протокол безопасного взаимодействия и исполнения между агентами. В текущей редакции протокол рассматривает `Arena` как пространство, где агенты собираются, общаются и совместно решают задачи, а также фиксирует обязательные механизмы отзыва ключей (`Key Revocation`) и ограничения ресурсов (`Resource Quotas`) для действий типа `execute`.

Основная спецификация находится в файле [VAPID_Core_Protocol_Standard_v1.0.md](./VAPID_Core_Protocol_Standard_v1.0.md).

## Что это

VAPID Core Protocol задаёт минимальный нормативный слой для доверенного обмена сообщениями и контролируемого выполнения задач между агентами.

Протокол определяет:

- что такое `Arena` как общее пространство совместной работы;
- как агенты распределяют роли и координируют выполнение задачи;
- как публикуется и проверяется статус ключей;
- как `Capability Token` ограничивает бюджет и ресурсы выполнения;
- в каком порядке `Receiver` обязан валидировать входящее сообщение;
- какие коды ошибок и тесты соответствия обязательны для совместимой реализации.

## Arena и роли агентов

`Arena` в VAPID Core Protocol это общий контекст, в котором несколько агентов работают над одной задачей, обмениваются сообщениями и артефактами и действуют в рамках общих ограничений безопасности и ресурсов.

Базовая ролевая модель включает:

- `orchestrator` — принимает задачу, разбивает её на шаги и управляет потоком работы;
- `worker` — выполняет назначенную подзадачу и возвращает результат;
- `auditor` — проверяет корректность, безопасность и полноту результата.

Типовой цикл работы внутри арены:

1. `orchestrator` принимает задачу и определяет границы арены.
2. `orchestrator` назначает подзадачи `worker`-агентам.
3. `worker`-агенты выполняют работу и обмениваются результатами.
4. `auditor` проверяет промежуточный или финальный результат.
5. `orchestrator` принимает итоговое решение по задаче.

## Ключевые свойства протокола

- `Arena` служит базовым пространством для координации агентов и маршрутизации сообщений.
- Роли агентов помогают разделять координацию, исполнение и аудит.
- Проверка отзыва ключей через `status_endpoint` перед критическими операциями.
- Ограничение бюджета выполнения через `resource_quota` в `Capability Token`.
- Атомарный учёт ресурсов в пределах `quota_scope`.
- Жёсткий `Receiver Pipeline`, который запрещает интерпретацию `body.data` до завершения проверок.
- Канонические коды ошибок для аутентификации, квот, политик и протокольных нарушений.
- Контрольный список соответствия для реализаций `Core Receiver` и `Secure Agent`.

## Статус

Этот репозиторий содержит первую цельную редакцию VAPID Core Protocol для базовой безопасной среды исполнения между агентами. Дополнительные возможности, такие как `Envelope Encryption`, `Billing & Settlement` и `P2P Profile`, рассматриваются как будущие расширения, а не как часть обязательного core.
